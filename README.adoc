== Prerequisites

Install the following before cloning:

* https://www.jetpack.io/devbox/docs/installing_devbox/[Devbox] — reproducible dev environment
* https://docs.docker.com/get-docker/[Docker] — runs the PostgreSQL database

== Setup

=== 1. Clone the repository

[source,bash]
----
git clone git@github.com:mikeyfennelly1/kingdom.git
cd kingdom
----

=== 2. Enter the devbox shell

All commands from this point forward must be run inside the devbox shell.

[source,bash]
----
devbox shell
----

=== 3. Configure environment variables

[source,bash]
----
cp .env.example .env
----

The defaults in `.env` work out of the box for local development. No changes needed unless you're running on a different host or port.

=== 4. Generate TLS certificate

The server requires a TLS certificate to start. Generate a self-signed certificate for local development:

[source,bash]
----
mkdir -p certs
openssl req -x509 -newkey rsa:4096 -sha256 -days 365 -nodes \
  -keyout certs/server.key \
  -out certs/server.crt \
  -subj "/CN=localhost" \
  -addext "subjectAltName=DNS:localhost,IP:127.0.0.1"
----

This creates `certs/server.crt` (public, committed to the repo) and `certs/server.key` (private, gitignored). If you pulled the repo and `certs/server.crt` already exists, you still need to generate your own `server.key`.

=== 5. Start the database

[source,bash]
----
docker compose up -d
----

PostgreSQL 16 will start on port 5432. The schema is created automatically when the server first starts.

== Building

[source,bash]
----
task build
----

Binaries are output to:

* `build/kds/kds` — Kingdom Data Server
* `build/kdctl/kdctl` — Kingdom Control CLI
* `build/libkd/libkd.dylib` — Shared library (macOS) / `libkd.so` (Linux)

== Running the Server

[source,bash]
----
task run
----

This loads environment variables from `.env` and starts `kds` on port 8080 over HTTPS. You should see:

----
[info] Initializing Kingdom Server...
[info] Connected to database
[info] Database schema initialized
[info] Starting Kingdom Server on 0.0.0.0:8080
----

To verify the server is running:

[source,bash]
----
curl --cacert certs/server.crt https://localhost:8080/health
----

Expected: `{"status":"ok"}`

== Using the CLI (kdctl)

All `kdctl` commands require `--protocol https` and `--ca-cert certs/server.crt` since the server uses TLS with a self-signed certificate. Set these as environment variables to avoid repeating them:

[source,bash]
----
export KD_PROTOCOL=https
export KD_CA_CERT=certs/server.crt
----

=== Interactive shell

[source,bash]
----
./build/kdctl/kdctl
----

Available shell commands:

----
signup              Create a new account
login               Log in
logout              Log out
health              Check server health
info                Server info
conversations       List your conversations
create-conversation Create a new conversation
send                Send a message
messages            List messages in a conversation
help                Show this list
exit                Quit
----

=== Subcommands

[source,bash]
----
./build/kdctl/kdctl health
./build/kdctl/kdctl signup -u alice -p hunter2
./build/kdctl/kdctl login -u alice -p hunter2
./build/kdctl/kdctl conversations -u <user-id>
./build/kdctl/kdctl create-conversation -n "My Chat" -p <user-id> -p <user-id>
./build/kdctl/kdctl send -c <conversation-id> -u <sender-id> -m "Hello"
./build/kdctl/kdctl messages -c <conversation-id>
----

== Blockchain Sidecar

The sidecar records message hashes on the Ethereum Sepolia testnet. It runs as a separate Node.js process alongside `kds`.

=== Setup

[source,bash]
----
cd blockchain/sidecar
cp ../.env.example .env
# Edit .env with SEPOLIA_RPC_URL, PRIVATE_KEY, CONTRACT_ADDRESS
npm install
npm start
----

The sidecar runs on port 3001 by default. The server (`kds`) calls it automatically when messages are sent via `KD_BLOCKCHAIN_SIDECAR_URL` (defaults to `http://localhost:3001`).

=== Verification page

Open `blockchain/verification/index.html` directly in a browser (no server needed):

[source,bash]
----
open blockchain/verification/index.html
----

Paste a transaction hash from a sent message and the message payload to verify on-chain integrity.

== Environment Variables

=== Server (`kds`)

[cols="1,1,2", options="header"]
|===
| Variable | Default | Description
| `KD_DB_URL` | — | PostgreSQL connection string (required)
| `KD_TLS_CERT` | — | Path to TLS certificate file (required)
| `KD_TLS_KEY` | — | Path to TLS private key file (required)
| `KD_PORT` | `8080` | Port to listen on
| `KD_LOG_LEVEL` | `info` | Log level (debug, info, warn, error)
| `KD_BLOCKCHAIN_SIDECAR_URL` | `http://localhost:3001` | Blockchain sidecar URL
|===

=== Client (`kdctl`)

[cols="1,1,2", options="header"]
|===
| Variable | Default | Description
| `KD_HOST` | `localhost` | Server host
| `KD_PORT` | `8080` | Server port
| `KD_PROTOCOL` | `http` | Protocol (`http` or `https`)
| `KD_CA_CERT` | — | Path to CA certificate for TLS verification
|===
