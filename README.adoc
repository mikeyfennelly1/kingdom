= Kingdom — Secure Messaging Application
:toc: left
:toclevels: 2

== Prerequisites

Install the following before cloning:

* https://www.jetpack.io/devbox/docs/installing_devbox/[Devbox] — reproducible dev environment
* https://docs.docker.com/get-docker/[Docker] — runs the PostgreSQL database

== Setup

1. Clones this repo.
2. Downloads nix.
3. Installs all dependencies.
4. Bootstraps environments for build and dev.
5. Builds the server (`kds`) and the Qt6 GUI client (`kdctl`).
6. Generates TLS certificates to run on the server.

[source, bash]
----
wget -qO- https://raw.githubusercontent.com/mikeyfennelly1/kingdom/main/scripts/install.sh | bash
----

== Generating TLS Certificates

[source,bash]
----
./scripts/init-certs.sh
----

This generates `certs/server.key` and `certs/server.crt` (self-signed, valid for `localhost`, `updakingdom.theburkenator.com`, `127.0.0.1`, and `200.69.13.70`).

== Running the Server

[source,bash]
----
task run
----

This loads environment variables from `.env` and starts `kds` on port 8080 over HTTPS. You should see:

----
[info] Initializing Kingdom Server...
[info] Connected to database
[info] Database schema initialized
[info] Starting Kingdom Server on 0.0.0.0:8080
----

To verify the server is running:

[source,bash]
----
./scripts/health-check.sh
----

Pass a custom base URL as the first argument if the server is not at the default (`https://updakingdom.theburkenator.com`):

[source,bash]
----
./scripts/health-check.sh https://localhost:8080
----

== Using the GUI (kdctl)

`kdctl` is a Qt6 desktop application. Launch it with:

[source,bash]
----
./build/kdctl/kdctl
----

=== Login screen

On startup you will see a login screen with three fields:

* **Server URL** — the address of the `kds` server, e.g. `https://localhost:8080`
* **Username**
* **Password**

Use **Sign Up** to register a new account or **Log In** to authenticate with an existing one. On first signup the client generates your X25519 and Ed25519 keypairs and uploads the public key bundle to the server. Your private keys are encrypted at rest under a key derived from your password.

=== Messaging

After login you will see the main window:

* **Left panel** — your conversation list. Click **New Conversation** to start one with another user.
* **Right panel** — messages in the selected conversation. Type in the box at the bottom and click **Send**.

All messages are end-to-end encrypted (X3DH + XChaCha20-Poly1305) before being sent to the server. The server never sees plaintext.

=== Additional actions

Select a message in the list to enable:

* **Forward** — re-encrypts and sends the message to another user or conversation
* **Delete** — removes the message from the server (sender only)
* **Revoke** — revokes another participant's access to the message

The conversation list and messages refresh automatically on a timer.

== Blockchain Sidecar

The sidecar records message hashes on the Ethereum Sepolia testnet. It runs as a separate Node.js process alongside `kds`.

=== Setup

[source,bash]
----
cd blockchain/sidecar
cp ../.env.example .env
# Edit .env with SEPOLIA_RPC_URL, PRIVATE_KEY, CONTRACT_ADDRESS, BLOCKCHAIN_SIDECAR_SECRET
npm install
npm start
----

The sidecar runs on port 3001 by default. The server (`kds`) calls it automatically when messages are sent via `KD_BLOCKCHAIN_SIDECAR_URL` (defaults to `http://localhost:3001`). Requests from `kds` to the sidecar are authenticated with `KD_BLOCKCHAIN_SIDECAR_SECRET`, which must match `BLOCKCHAIN_SIDECAR_SECRET` in the sidecar environment.

=== Verification page

Open `blockchain/verification/index.html` directly in a browser (no server needed):

[source,bash]
----
open blockchain/verification/index.html
----

Paste a transaction hash from a sent message and the message payload to verify on-chain integrity.

== Environment Variables

=== Server (`kds`)

[cols="1,1,2", options="header"]
|===
| Variable | Default | Description
| `KD_DB_URL` | — | PostgreSQL connection string (required)
| `KD_TLS_CERT` | — | Path to TLS certificate file (required)
| `KD_TLS_KEY` | — | Path to TLS private key file (required)
| `KD_JWT_SECRET` | — | Secret used to sign HS256 authentication tokens (required, at least 32 chars)
| `KD_BLOCKCHAIN_SIDECAR_SECRET` | — | Shared secret sent to the blockchain sidecar (required, at least 32 chars)
| `KD_JWT_TTL_SECONDS` | `3600` | Authentication token lifetime in seconds
| `KD_PORT` | `8080` | Port to listen on
| `KD_LOG_LEVEL` | `info` | Log level (debug, info, warn, error)
| `KD_BLOCKCHAIN_SIDECAR_URL` | `http://localhost:3001` | Blockchain sidecar URL
| `KD_RATE_LIMIT_MAX` | `10` | Max auth attempts per IP per 60-second window
|===

=== Blockchain sidecar

[cols="1,1,2", options="header"]
|===
| Variable | Default | Description
| `SEPOLIA_RPC_URL` | — | Sepolia RPC endpoint (required)
| `PRIVATE_KEY` | — | Ethereum private key used to submit batch transactions (required)
| `CONTRACT_ADDRESS` | — | Deployed Sepolia contract address (required)
| `BLOCKCHAIN_SIDECAR_SECRET` | — | Shared secret required on `/record` and `/pending/:id` (required, at least 32 chars)
|===
